## Design Context

### Users
Software developers who use AI-assisted coding tools (Claude Code, Copilot, Cursor, etc.). They range from beginners just starting with AI autocomplete to power users running autonomous agent orchestration. They arrive to self-assess where they stand, discover what's next, and share their profile. Context is professional but informal — they're curious, not filling out a compliance form.

### Brand Personality
**Playful, epic, approachable.**

- Voice: encouraging, never condescending. Speaks like a teammate, not a teacher.
- Tone: light and energetic for beginners ("you're on your way!"), celebratory for advanced users ("you're running autonomous orchestration — legendary").
- The RPG framing is genuine, not ironic — lean into it with conviction.

### Emotional Goals
- **Beginners**: motivation to level up. The tree should feel like an exciting progression, not an intimidating checklist. Every level reached is a win.
- **Experienced users**: pride and shareability. Seeing high-level nodes claimed should feel awesome — worthy of sharing. The "I've reached this level" moment should feel like earning a badge.
- **Everyone**: curiosity to explore what's ahead. Locked nodes should intrigue, not gatekeep.

### Aesthetic Direction
- **Visual tone**: Playful RPG meets clean developer tool. Not cartoonish, not corporate.
- **Theme**: Light mode only (warm cream `#faf7f2` base). No dark mode toggle for now.
- **References**: [plepic.com](https://plepic.com) and [plepic.com/training](https://plepic.com/training) — the parent brand. Match warmth, green identity, and confident simplicity.
- **Anti-references**: Generic SaaS dashboards. Gray-on-gray admin panels. Overly complex skill matrices or spreadsheet-like assessment tools.
- **The tree is the hero** — minimal chrome, maximum focus on the skill map itself.

### Design System
Plepic design system (locked palette, 2026-04-01):
- **Green system**: H=137°, S=73%. Surface `#edfcf1` through dark `#0d5822`. Brand green `#137b30` for headings/links.
- **Accent orange**: `#e26c45` — ONE per viewport. CTAs and urgency only, never decorative.
- **Neutrals**: Warm cream bg, white surface cards, near-black text.
- **Fonts**: Zilla Slab (display/headings), Plus Jakarta Sans (body/UI), JetBrains Mono (code/labels).
- **Spacing**: Tight groupings + generous separations. Related elements close, sections far apart.
- **Accessibility**: WCAG AA standard. Contrast ratios already validated in design system tokens.

### Design Principles

1. **The tree is the game** — The skill tree IS the interface, not a visualization bolted onto a form. Every interaction happens in context of the map.
2. **Progress feels rewarding** — Claiming a level should have visual weight: glow, color shift, distinction. The UI celebrates advancement.
3. **Peek ahead, never gatekeep** — Locked levels are visible and explorable. Curiosity is encouraged. Information is never hidden behind progression.
4. **Less chrome, more content** — Minimal navigation, no sidebars, no dashboards. The tree fills the screen. UI elements serve the tree, not the other way around.
5. **Shareable by design** — Profiles should look good enough that users want to screenshot or link them. The shared view is a first-class experience, not a degraded one.
